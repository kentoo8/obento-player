# コミットメッセージのフォーマット

prefix + 日本語で記載すること。

## prefixの例

- feat: 新機能
- fix: バグ修正
- docs: ドキュメント修正
- style: フォーマット修正など
- refactor: リファクタリング
- perf: パフォーマンス改善
- test: テスト関連
- chore: ビルド、補助ツール、ライブラリ関連

# 運用ルール
- 機能の変更をした際は、必ず`README.md`をチェックし、同期すること。
- 勝手に `git commit` を行うことは禁止。必ずユーザーの明示的な指示があった場合のみ行うこと。
- `git push` はどのような場合であっても絶対に行わないこと。
