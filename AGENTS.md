# コミットメッセージのフォーマット

prefix + 日本語で記載すること。

## prefixの例

- feat: 新機能
- fix: バグ修正
- docs: ドキュメント修正
- style: フォーマット修正など
- refactor: リファクタリング
- perf: パフォーマンス改善
- test: テスト関連
- chore: ビルド、補助ツール、ライブラリ関連

## デザイン & 実装ルール
- **アプリ名**: Obento Player（お弁当のような配置で動画を観るというコンセプト）
- **言語設定**: 静的なUI表示（ロゴや単位等）は英語、ツールチップや選択肢等のポップアップ要素は日本語とする。
- **用語**: 「スキップ」ではなく「除外」を使用する。
- **コミット**: UIの変更や調整は `feat` を使用する（`style` はコード整形のみに使用）。
- **Git操作**: 勝手に `git push` を行うのは絶対に禁止。プッシュは必ずユーザーの明示的な指示があった場合のみ行うこと。
